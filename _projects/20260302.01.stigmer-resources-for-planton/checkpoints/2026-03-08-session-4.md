## Decisions Made

- **Script numbering**: Used 05 and 06 (next available pair after 00, 01, 03, 04)
- **Read+write tool profile**: Cloud Resource Assistant agent needs both read AND write MCP tools (create, update, apply), unlike Infra Chart Composer which is read-only. This distinction is explicitly stated in the agent draft prompt so the agent-creator selects the right tools.
- **Preset-first workflow**: Skill prompt emphasizes always checking Cloud Object Presets before building specs from scratch — this is central to the Cloud Resource Assistant's workflow and absent from Infra Chart Composer.
- **Schema discovery**: Same MCP resource URIs as Infra Chart Composer (`cloud-resource-kinds://catalog`, `cloud-resource-schema://{kind}`) but used for direct spec field authoring rather than template validation.
- **Resource ID pattern**: Skill prompt includes the `<kind-prefix>-<org>-<name>` pattern with 27-char limit — critical domain knowledge for generating valid manifests.

## Key Code Changes

- `tools/05_draft-cloud-resource-assistant-skill.sh`: Follows the same shell boilerplate as `03_draft-infra-chart-composer-skill.sh`. Prompt tells the skill-creator to explore what-is-* docs (Cloud Resources, Kinds, Objects, Presets, OpenMCF), production YAML manifests, changelogs, and proto APIs. Emphasizes schema discovery, preset-first workflow, and ID patterns.
- `tools/06_draft-cloud-resource-assistant-agent.sh`: Follows the same shell boilerplate as `04_draft-infra-chart-composer-agent.sh`. Key difference: prompt explicitly instructs agent-creator to select read+write tools and include destructive tools covered by approval policy.

## Learnings

- The pattern established by Phase 4 (separate skill + agent scripts, discovery-oriented prompts, conventions rule) scales cleanly to new agents. Phase 2 scripts required only domain-specific prompt changes, not structural changes.
- The key differentiator between agents is their tool access profile (read-only vs read+write) and their relationship to presets/schemas.
